# Expression

## 概要

Cypherにおける式（Expressions）は、値の計算、演算の実行、クエリ内でのデータ操作を可能にする基本的な構成要素です。リレーショナルデータ操作に焦点を当てた従来のSQLとは異なり、Cypherの式はグラフデータ構造専用に設計されており、リレーションシップのトラバース、パターンマッチング、グラフ特有の計算を強力にサポートします。

Cypherの式には以下の主要な用途があります：

- **値の計算**：ノードのプロパティ、リレーションシップの属性、または計算結果から派生値を算出  
- **条件評価**：ノード、リレーションシップ、パスのフィルタリングのためのboolean式を作成  
- **データ変換**：表示またはさらなる処理のためにデータ型を変換・フォーマット  
- **グラフトラバーサル**：パス式とパターンマッチングを使用してグラフ構造をナビゲート  
- **集約**：ノードまたはリレーションシップのコレクションに対して計算を実行  
- **文字列およびテキスト処理**：検索、表示、分析のためのテキストデータの操作  

Cypherの式は、以下の2つの主要なカテゴリのコンポーネントから構成されます：

1. **演算子（Operators）**：オペランドに対して基本的な操作を実行する記号およびキーワード  
2. **関数（Functions）**：複雑なロジックをカプセル化し、パラメータを受け取る事前定義された操作  

これらのコンポーネントが組み合わさることで、単純な計算から複雑なグラフ操作まで、幅広い用途に対応する強力な式を構築できます。

## 演算子 (Operators)

Neug における演算子は、オペランドに対して操作を行う記号またはキーワードです。これらは式やクエリを構成する基本的な要素です。

### サポートされている演算子の種類

| 種類 | 説明 |
|------|-------------|
| [Comparison](./expression/comparison_op.md) | 値を比較するための演算子 (例: `=`, `<>`, `<`, `>`, `<=`, `>=`) |
| [Logical](./expression/logical_op.md) | 条件を組み合わせるためのブール演算子 (例: `AND`, `OR`, `NOT`) |
| [Arithmetic](./expression/arithmetic_op.md) | 数学的演算 (例: `+`, `-`, `*`, `/`, `%`) |
<!-- | Bit | ビット演算 (例: `&`, `|`, `^`, `<<`, `>>`) | -->
| [Null](./expression/null_op.md) | null 値を扱うための演算子 (例: `IS NULL`, `IS NOT NULL`) |
| [List](./expression/list_op.md) | リストデータ構造を操作するための演算子 (例: `IN`) |

## 関数

Neug における関数は、入力パラメータを受け取り、計算された値を返すための事前定義された操作です。これらはデータ操作および分析のための特殊な機能を提供します。

### サポートされている関数カテゴリ

| カテゴリ | 説明 |
|----------|-------------|
| [集約](./expression/agg_func.md) | 値のコレクションに対して操作を行い、単一の結果を返す関数（例: `COUNT`、`SUM`、`AVG`、`MAX`、`MIN`） |
| [キャスト](./expression/cast_func.md) | 異なるフォーマット間でデータ型を変換するための関数 |
| [時間](./expression/temporal_func.md) | 日付および時刻データを扱うための関数 |
| [グラフパターン](./expression/graph_func.md) | ノード、リレーションシップ、またはパス用に特別に設計された関数 |
