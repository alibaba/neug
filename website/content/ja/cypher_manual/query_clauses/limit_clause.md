# Limit 句

Limit は出力結果の数を制御するために使用されます。単独で使用するだけでなく、Order BY と組み合わせて TopK 操作にも使用できます。Neug は現在、Limit で2種類の式をサポートしています：1. 単一の Integer 定数。2. 定数パラメータのみを持つ算術式。

## Integer 値での Limit

```
Match (a:person)
Return a.age
Limit 2;
```
出力に順序がないため、結果は任意の2つの結果になる可能性があります。

出力:
```
+------------+
|   _0_a.age |
+============+
|         29 |
+------------+
|         27 |
+------------+
```

## Integer 式での Limit

```
Match (a:person)
Return a.age
Limit 1+1;
```

出力:
```
+------------+
|   _0_a.age |
+============+
|         29 |
+------------+
|         27 |
+------------+
```

# Skip 句

Limit は出力結果の数を制御し、これは出力結果の行番号範囲を [0, upper_bound) とすることに相当します。Skip は出力結果の最初の数行をスキップすることを制御し、これは出力結果の行番号範囲を [lower_bound, +∞) とすることに相当します。ここで、行番号は 0 から始まると仮定します。

## 整数値での Skip

```
Match (a:person)
Return a.age
Skip 2;
```
このクエリは、結果の最初の2行をスキップするために使用されます。

出力:
```
+------------+
|   _0_a.age |
+============+
|         32 |
+------------+
|         35 |
+------------+
```

## 整数式での Skip

```
Match (a:person)
Return a.age
Skip 1+1;
```

出力:
```
+------------+
|   _0_a.age |
+============+
|         32 |
+------------+
|         35 |
+------------+
```
