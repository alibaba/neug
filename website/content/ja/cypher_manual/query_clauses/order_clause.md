# Order 句

Order は、現在の結果をプロパティに基づいてソートし、確定的な出力を保証するために使用されます。現在サポートしているソートオプションは2つあります：`ASC` は昇順、`DESC` は降順です。特に指定しない場合、デフォルトは昇順です。注目すべきは、Order は Limit と組み合わせて使用でき、これは TopK 操作と同等であることです。以下に一般的な使用パターンを紹介します。

## 単一プロパティによる Order

```
Match (a)
Return a.name
ORDER BY a.name ASC;
```

出力:
```
+-------------+
| _0_a.name   |
+=============+
| josh        |
+-------------+
| lop         |
+-------------+
| marko       |
+-------------+
| peter       |
+-------------+
| ripple      |
+-------------+
| vadas       |
+-------------+
```

## 複数のプロパティでソート

```
Match (a)-[b]->(c)
Return a.name, b.weight
ORDER BY a.name ASC, b.weight ASC;
```

出力:
```
+-------------+---------------+
| _0_a.name   |   _4_b.weight |
+=============+===============+
| josh        |           0.4 |
+-------------+---------------+
| josh        |           1   |
+-------------+---------------+
| marko       |           0.4 |
+-------------+---------------+
| marko       |           0.5 |
+-------------+---------------+
| marko       |           1   |
+-------------+---------------+
| peter       |           0.2 |
+-------------+---------------+
```

## 式によるソート

プロパティによる直接的なソートに加えて、ORDER BY句にはより複雑な式も指定できます。例えば、算術演算の結果やスカラー関数の戻り値などが該当します。

### 事前計算式によるソート

```
Match (a)-[b]->(c)
Return a.age, c.name
ORDER BY a.age + 10 ASC, c.name;
```

出力:
```
+------------+-------------+
|   _0_a.age | _2_c.name   |
+============+=============+
|         29 | josh        |
+------------+-------------+
|         29 | lop         |
+------------+-------------+
|         29 | vadas       |
+------------+-------------+
|         32 | lop         |
+------------+-------------+
|         32 | ripple      |
+------------+-------------+
|         35 | lop         |
+------------+-------------+
```

### スカラー関数の結果によるソート

```
Match (a)-[b]->(c)
Return a.name, c.name
Order BY label(a);
```

## Limit 付きの Order

さらに、BI (Business Intelligence) クエリのシナリオでは、TopK が最も一般的な操作の一つであり、最も重要な結果のみを切り詰めて出力します。Neug はこのようなクエリもサポートしています。

```
Match (a)-[b]->(c)
Return a.age, c.name
ORDER BY a.age + 10 ASC, c.name ASC
Limit 2;
``` 

出力:
```
+------------+-------------+
|   _0_a.age | _2_c.name   |
+============+=============+
|         29 | josh        |
+------------+-------------+
|         29 | lop         |
+------------+-------------+
```
