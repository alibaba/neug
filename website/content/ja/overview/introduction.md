# NeuG のご紹介

NeuG（発音は「new-gee」）へようこそ。NeuG は HTAP（Hybrid Transactional/Analytical Processing）グラフデータベースで、高性能な組み込みグラフ分析とリアルタイムトランザクション処理向けに設計されています。

NeuG は、運用系のグラフワークロードと複雑な分析クエリの両方を同じデータ上で処理できるシステムに対する日益増加するニーズに対応するため構築されており、最新のグラフアプリケーション向けの統合プラットフォームを提供します。

## NeuG とは？

NeuG は、**独自のデュアルモードアーキテクチャ**により、以下2つの世界の長所を組み合わせたグラフデータベースです：
- **トランザクション処理 (TP)**: ACID保証付きのリアルタイムグラフ操作
- **分析処理 (AP)**: 高性能なグラフ分析とパターンマッチング
- **デュアル接続モード**: 組み込み型またはサービスベースのシステムとして柔軟にデプロイ可能
- **最適化されたパフォーマンス**: 各モードは対象ワークロードに特化してチューニング

モダンな C++ コアエンジン上に構築され、NeuG は以下を提供します：
- Cypher クエリ言語を備えたプロパティグラフデータモデル
- 圧縮された隣接リストによるカラムナ形式のディスクベースストレージ
- ベクトル化・並列化されたクエリ処理
- MVCC ベースのトランザクション管理

## なぜ NeuG を選ぶのか？

### 独自のデュアルモードアーキテクチャ

NeuG の最大の特徴は、異なるグラフワークロードパターンに対してパフォーマンスを最適化する **デュアルモード設計** です：

#### 埋め込みモード - Analytics用に最適化 (AP)
**最適な用途**: 複雑なパターンマッチング、全グラフ計算、分析ワークロード
- **バッチデータローディング**: 大規模データインポートを効率的に処理
- **最大クエリパフォーマンス**: 複雑なクエリのために利用可能なすべてのCPUコアを活用
- **フルリソース利用**: すべてのクエリが完全なシステムリソースを活用可能

**典型的なユースケース**:
- グラフ分析およびデータサイエンスワークロード
- 複雑なパターン検出およびグラフアルゴリズム
- バッチ処理およびETL操作
- 研究および探索的データ分析

#### Service Mode - トランザクション向けに最適化 (TP)
**最適な用途**: 頻繁な小規模な更新と同時アクセスがあるリアルタイムアプリケーション
- **同時読み書き操作**: 高頻度で小規模なデータ変更に最適化
- **ポイントクエリ**: グラフデータの小規模なサブセットへの効率的なアクセス
- **マルチセッション対応**: 複数の同時接続を処理
- **低レイテンシ操作**: トランザクションワークロードに対する高速な応答時間

**典型的な利用ケース**:
- リアルタイム推薦システム
- 不正検知とリスク管理
- オンライントランザクション処理

### フレキシブルなモードの組み合わせ

NeuG の真の力は、両方のモードを**柔軟に組み合わせる**ことにある：

- **Data Pipeline 統合**: 初期の一括データロードには embedded mode を使用し、運用クエリには service mode に切り替える
- **ハイブリッドワークフロー**: 実行中の TP サービスからチェックポイントされたデータをエクスポートし、集中的な AP 分析用に別のコピーを作成する
- **開発から本番環境へ**: embedded mode で開発・テストを行い、本番環境では service mode としてデプロイする
- **ワークロードの分離**: 専用の embedded インスタンスで AP ワークロードを実行しながら、TP サービスを維持する

> **Note**: embedded mode は小規模な変更には対応できるが、頻繁な書き込みは排他ロック設計のためすべての読み取りクエリをブロックする（逆も同様）。同様に、service mode は大規模なパターンクエリを実行できるが、すべての利用可能なスレッドを単一クエリの高速化に使用することは推奨されない。これは他の同時実行クエリのリソースを奪うことになるためである。

### パフォーマンスとスケーラビリティ
NeuG は以下のような高性能グラフワークロード向けに設計されています：
- 高度なクエリ最適化と実行計画
- 分析ワークロード向けのベクトル化演算
- グラフ走査に最適化された効率的なストレージ形式
- 複雑なクエリに対応するマルチコア並列処理

### 柔軟性と使いやすさ
- **クロスプラットフォーム**: x86 および ARM アーキテクチャの Linux および macOS で動作
- **簡単な統合**: 組み込みアーキテクチャによりサーバー管理のオーバーヘッドを排除
- **モード切替**: 組み込みモードとサービスモードをシームレスに切り替え可能

### エンタープライズ向け機能
- **ACID Transactions**: シリアル化可能なACID保証により、ミッションクリティカルなアプリケーションに対応
- **Concurrent Access**: 読み取り重視の分析と書き込み重視の操作の両方に対して最適化
- **Error Handling**: 包括的なエラー管理およびリカバリメカニズム
- **Monitoring**: 組み込みのパフォーマンス監視およびクエリプロファイリング

NeuGは、グラフ計算および大規模データ処理の専門知識で知られるアリババの主要研究グループである[GraphScope](https://graphscope.io)チームによって開発されました。このフレームワークは、アリババがEC、サプライチェーン最適化、不正検出などの分野でエンタープライズ規模の課題を解決するためにグラフ分析を展開するという豊富な経験から得られた高度な手法を統合しています。
