# Limit 子句

Limit 用于控制输出结果的数量。除了单独使用外，还可以与 Order BY 一起使用进行 TopK 操作。NeuG 目前在 Limit 中支持两种类型的表达式：1. 单个 Integer 常量。2. 只包含常量参数的算术表达式。

## 使用整数值的 Limit

```
Match (a:person)
Return a.age
Limit 2;
```
由于输出没有排序，结果可能是任意两个记录。

输出：
```
+------------+
|   _0_a.age |
+============+
|         29 |
+------------+
|         27 |
+------------+
```

## 使用整数表达式的 Limit

```
Match (a:person)
Return a.age
Limit 1+1;
```

输出：
```
+------------+
|   _0_a.age |
+============+
|         29 |
+------------+
|         27 |
+------------+
```

# Skip 子句

Limit 用于控制输出结果的数量，相当于确定输出结果的行号范围为 [0, upper_bound)。Skip 用于控制跳过输出结果的前几行，相当于确定输出结果的行号范围为 [lower_bound, +∞)，这里我们假设行号从 0 开始。

## 使用整数值的 Skip

```
Match (a:person)
Return a.age
Skip 2;
```
该查询用于跳过结果的前两行。

输出：
```
+------------+
|   _0_a.age |
+============+
|         32 |
+------------+
|         35 |
+------------+
```

## 使用整数表达式的 Skip

```
Match (a:person)
Return a.age
Skip 1+1;
```

输出：
```
+------------+
|   _0_a.age |
+============+
|         32 |
+------------+
|         35 |
+------------+
```
