# Order 子句

Order 用于根据属性对当前结果进行排序，以确保输出的确定性。我们目前支持两种排序选项：`ASC` 表示升序，`DESC` 表示降序。如果没有特别指定，默认为升序。值得注意的是，Order 可以与 Limit 结合使用，这相当于 TopK 操作。我们将介绍以下常见的使用模式。

## 按单个属性排序

```
Match (a)
Return a.name
ORDER BY a.name ASC;
```

输出：
```
+-------------+
| _0_a.name   |
+=============+
| josh        |
+-------------+
| lop         |
+-------------+
| marko       |
+-------------+
| peter       |
+-------------+
| ripple      |
+-------------+
| vadas       |
+-------------+
```

## 按多个属性排序

```
Match (a)-[b]->(c)
Return a.name, b.weight
ORDER BY a.name ASC, b.weight ASC;
```

输出：
```
+-------------+---------------+
| _0_a.name   |   _4_b.weight |
+=============+===============+
| josh        |           0.4 |
+-------------+---------------+
| josh        |           1   |
+-------------+---------------+
| marko       |           0.4 |
+-------------+---------------+
| marko       |           0.5 |
+-------------+---------------+
| marko       |           1   |
+-------------+---------------+
| peter       |           0.2 |
+-------------+---------------+
```

## 按表达式排序

除了直接按属性排序外，ORDER BY 子句还可以使用更复杂的表达式，例如算术运算结果、标量函数返回值等。

### 按预计算表达式排序

```
Match (a)-[b]->(c)
Return a.age, c.name
ORDER BY a.age + 10 ASC, c.name;
```

输出：
```
+------------+-------------+
|   _0_a.age | _2_c.name   |
+============+=============+
|         29 | josh        |
+------------+-------------+
|         29 | lop         |
+------------+-------------+
|         29 | vadas       |
+------------+-------------+
|         32 | lop         |
+------------+-------------+
|         32 | ripple      |
+------------+-------------+
|         35 | lop         |
+------------+-------------+
```

### 按标量函数结果排序

```
Match (a)-[b]->(c)
Return a.name, c.name
Order BY label(a);
```

## 限制结果数量的排序

此外，在 BI (Business Intelligence) 查询场景中，TopK 是最常见的操作之一，只截取并输出最重要的结果。NeuG 也支持此类查询。

```
Match (a)-[b]->(c)
Return a.age, c.name
ORDER BY a.age + 10 ASC, c.name ASC
Limit 2;
``` 

输出:
```
+------------+-------------+
|   _0_a.age | _2_c.name   |
+============+=============+
|         29 | josh        |
+------------+-------------+
|         29 | lop         |
+------------+-------------+
```
