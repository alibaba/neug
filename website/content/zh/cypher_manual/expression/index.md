# 表达式

表达式是 Cypher 中的基础组件，允许你在查询中计算值、执行运算和操作数据。与专注于关系数据操作的传统 SQL 不同，Cypher 表达式专为图数据结构设计，提供了强大的遍历边、模式匹配和图特定计算的能力。

Cypher 表达式有以下几个主要用途：

- **值计算**：从节点属性、边属性或计算结果中派生值
- **条件评估**：创建布尔表达式用于过滤节点、边或路径
- **数据转换**：转换和格式化数据类型以供显示或进一步处理
- **图遍历**：使用路径表达式和模式匹配在图结构中导航
- **聚合**：对节点或边的集合执行计算
- **字符串和文本处理**：操作文本数据以进行搜索、显示或分析


Cypher 表达式由两类主要组件构成：

1. **操作符**：对操作数执行基本操作的符号和关键字
2. **函数**：封装复杂逻辑并接受参数的预定义操作

这些组件协同工作，创建出既能处理简单计算又能处理复杂图操作的强大表达式。

## 运算符

NeuG 中的运算符是用于对操作数执行操作的符号或关键字。它们是构建表达式和查询的基本组成部分。

### 支持的运算符类型

| 类型 | 描述 |
|------|-------------|
| [比较运算符](comparison_op) | 用于比较值的运算符（例如 `=`, `<>`, `<`, `>`, `<=`, `>=`） |
| [逻辑运算符](logical_op) | 用于组合条件的布尔运算符（例如 `AND`, `OR`, `NOT`） |
| [算术运算符](arithmetic_op) | 数学运算（例如 `+`, `-`, `*`, `/`, `%`） |
| [空值运算符](null_op) | 处理空值的操作（例如 `IS NULL`, `IS NOT NULL`） |
| [列表运算符](list_op) | 处理列表数据结构的操作（例如 `IN`） |

## Functions

NeuG 中的 Functions 是预定义的操作，接受输入参数并返回计算后的值。它们为数据处理和分析提供了专门的功能。

### 支持的 Function 分类

| 分类 | 描述 |
|----------|-------------|
| [Aggregate](agg_func) | 对值集合进行操作并返回单一结果的函数（例如 `COUNT`、`SUM`、`AVG`、`MAX`、`MIN`） |
| [Cast](cast_func) | 用于在不同格式之间转换数据类型的函数 |
| [Temporal](temporal_func) | 用于处理日期和时间数据的函数 |
| [Graph Pattern](graph_func) | 专门为 nodes、edges 或 path 设计的函数 |
