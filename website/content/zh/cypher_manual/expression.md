# Expression

## 概述

表达式（Expressions）是 Cypher 中的基础组件，允许你在查询中计算值、执行运算以及操作数据。与专注于关系型数据操作的传统 SQL 不同，Cypher 表达式专为图数据结构设计，提供了强大的能力来遍历关系、进行模式匹配以及执行图特有的计算。

Cypher 表达式主要有以下几个用途：

- **值计算**：基于节点属性、关系属性或计算结果生成派生值  
- **条件判断**：创建布尔表达式，用于过滤节点、关系或路径  
- **数据转换**：转换和格式化数据类型，便于展示或进一步处理  
- **图遍历**：使用路径表达式和模式匹配在图结构中导航  
- **聚合操作**：对节点或关系集合执行计算  
- **字符串与文本处理**：操作文本数据，用于搜索、展示或分析  

Cypher 表达式由两类主要组件构成：

1. **操作符（Operators）**：对操作数执行基本操作的符号和关键字  
2. **函数（Functions）**：封装复杂逻辑的预定义操作，可接受参数  

这些组件协同工作，构建出既能处理简单计算，又能执行复杂图操作的强大表达式。

## 运算符

## Functions

Neug 中的 Functions 是预定义的操作，接受输入参数并返回计算后的值。它们为数据处理和分析提供了专门的功能。

### 支持的 Function 分类

| 分类 | 描述 |
|----------|-------------|
| [Aggregate](./expression/agg_func.md) | 对值集合进行操作并返回单一结果的函数（例如 `COUNT`、`SUM`、`AVG`、`MAX`、`MIN`） |
| [Cast](./expression/cast_func.md) | 用于在不同格式之间转换数据类型的函数 |
| [Temporal](./expression/temporal_func.md) | 用于处理日期和时间数据的函数 |
| [Graph Pattern](./expression/graph_func.md) | 专门为节点、关系或路径设计的函数 |
