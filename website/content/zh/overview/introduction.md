# NeuG 介绍

欢迎使用 NeuG（发音为 "new-gee"），这是一款 HTAP（混合事务/分析处理）图数据库，专为高性能嵌入式图分析和实时事务处理而设计。

NeuG 旨在满足日益增长的系统需求，能够同时处理操作型图工作负载和同一数据上的复杂分析查询，为现代图应用提供统一平台。

## 什么是 NeuG？

NeuG 是一个图数据库，通过其**独特的双模式架构**结合了两个世界的优势：
- **事务处理 (TP)**：具有 ACID 保证的实时图操作
- **分析处理 (AP)**：高性能图分析和模式匹配
- **双连接模式**：作为嵌入式或基于服务的系统灵活部署
- **优化性能**：每种模式都针对其目标工作负载进行了专门调优

基于现代 C++ 核心引擎构建，NeuG 提供：
- 支持 Cypher 查询语言的属性图数据模型
- 基于磁盘的列式存储，采用压缩邻接表
- 向量化和并行化查询处理
- 基于 MVCC 的事务管理

## 为什么选择 NeuG？

### 独特的双模式架构

NeuG 的突出特性是其**双模式设计**，可为不同的图工作负载模式优化性能：

#### 嵌入模式 - 为分析优化 (AP)
**最适合**：复杂模式匹配、全图计算和分析工作负载
- **批量数据加载**：高效处理大规模数据导入
- **最大查询性能**：利用所有可用 CPU 核心处理复杂查询
- **_full 资源利用**：每个查询都可以利用完整的系统资源

**典型使用场景**：
- 图分析和数据科学工作负载
- 复杂模式检测和图算法
- 批处理和 ETL 操作
- 研究和探索性数据分析

#### 服务模式 - 针对事务优化 (TP)
**最适合**：需要频繁小规模更新和并发访问的实时应用
- **并发读写操作**：针对高频、小规模数据修改进行了优化
- **点查询**：高效访问图数据的小子集
- **多会话支持**：处理多个并发连接
- **低延迟操作**：为事务性工作负载提供快速响应时间

**典型使用场景**：
- 实时推荐系统
- 欺诈检测和风险管理
- 在线事务处理

### 灵活的模式组合

NeuG 的真正强大之处在于两种模式的**灵活组合**：

- **数据管道集成**：使用嵌入式模式进行初始批量数据加载，然后切换到服务模式进行操作查询
- **混合工作流**：从运行中的 TP 服务导出检查点数据到独立副本，用于密集的 AP 分析
- **从开发到生产**：使用嵌入式模式进行开发和测试，部署为生产环境的服务模式
- **工作负载分离**：在专用的嵌入式实例上运行 AP 工作负载，同时维护 TP 服务

> **注意**：虽然嵌入式模式可以处理小规模修改，但由于其独占锁设计，频繁写入会阻塞所有读取查询（反之亦然）。同样，服务模式可以执行大规模模式查询，但不建议将所有可用线程都用于单个查询加速，因为这会导致其他并发查询资源不足。

### 性能与可扩展性
NeuG 专为高性能图计算工作负载而设计，具备以下特性：
- 高级查询优化和执行计划
- 面向分析型工作负载的向量化操作
- 针对图遍历优化的高效存储格式
- 支持复杂查询的多核并行处理

### 灵活性与易用性
- **跨平台支持**：支持在 x86 和 ARM 架构的 Linux 与 macOS 系统上运行
- **易于集成**：嵌入式架构，无需管理服务器开销
- **模式切换**：可在嵌入模式与服务模式之间无缝切换

### 企业级特性
- **ACID 事务**：为关键业务应用提供可序列化的 ACID 保证
- **并发访问**：针对读密集型分析和写密集型操作进行了优化
- **错误处理**：全面的错误管理与恢复机制
- **监控**：内置性能监控和查询性能分析功能
