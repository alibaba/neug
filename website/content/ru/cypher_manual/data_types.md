## Подробное введение

### Примитивные типы

#### INT32
- **Описание**: 32-битный знаковый целочисленный тип
- **Диапазон**: [2,147,483,648, 2,147,483,647]
- **Пример запроса**: `Return CAST(42, 'INT32') as int32_value;`

#### UINT32
- **Описание**: 32-битный беззнаковый целочисленный тип
- **Диапазон**: [0, 4,294,967,295]
- **Пример запроса**: `RETURN CAST(42, 'UINT32') AS uint32_value;`

#### INT64
- **Описание**: 64-битный знаковый целочисленный тип, тип по умолчанию для целочисленных значений
- **Диапазон**: [-9,223,372,036,854,775,808, 9,223,372,036,854,775,807]
- **Пример запроса**: `RETURN 9223372036854775807 AS int64_value;`

#### UINT64
- **Описание**: 64-битный беззнаковый целочисленный тип
- **Диапазон**: [0, 18,446,744,073,709,551,615]
- **Пример запроса**: `RETURN CAST(18446744073709551615, 'UINT64') AS uint64_value;`

#### FLOAT
- **Описание**: Число с плавающей точкой одинарной точности
- **Точность**: ~7 десятичных цифр
- **Пример запроса**: `RETURN CAST(3.14, 'FLOAT') AS float_value;`

#### DOUBLE
- **Описание**: Число с плавающей точкой двойной точности, тип по умолчанию для значений float
- **Точность**: ~15-17 десятичных цифр
- **Пример запроса**: `RETURN 3.14159265359 AS double_value;`

#### BOOL
- **Описание**: Логический тип, представляющий значения true или false
- **Значения**: `true`, `false`
- **Пример запроса**: `RETURN true AS bool_value;`

#### NULL
- **Описание**: Представляет отсутствующие или неопределенные значения
- **Пример запроса**: `RETURN null AS null_value;`

### Типы строк

#### VARCHAR
- **Описание**: Строка переменной длины с кодировкой UTF-8
- **Пример запроса**: `RETURN 'Hello World' AS string_value;`
- **Длина**: Переменная, ограничена системными ограничениями, по умолчанию `65536`

### Временные типы

#### DATE
- **Описание**: Тип даты для хранения календарных дат
- **Формат**: YYYY-MM-DD
- **Пример запроса**: `RETURN date('2022-06-06') AS date_value;`

#### DATETIME
- **Описание**: Тип, объединяющий дату и время
- **Формат**: YYYY-MM-DD HH:MM:SS
- **Пример запроса**: `RETURN timestamp('2022-06-06 12:00:00') AS datetime_value;`

#### INTERVAL
- **Описание**: Тип длительности или временного интервала, в соответствии со [спецификацией kuzu]().
- **Пример запроса**: `RETURN interval('1 year 2 days') AS interval_value;`

### Составные типы

#### LIST
- **Описание**: Упорядоченная коллекция значений с гетерогенными типами
- **Пример запроса**: `RETURN [1, 2, 3] AS list_value;`

В следующей таблице представлены все типы компонентов, которые поддерживает LIST:

| Категория | Тип | Пример |
|-----------|-----|--------|
| Числовой | INT32, INT64, UINT32, UINT64, DOUBLE, FLOAT | `RETURN [1, 2, 3.0];` |
| Строка | VARCHAR | `RETURN ['marko', 'josh'];` |
| Дата | DATE, DATETIME | `RETURN [date('2011-01-25'), timestamp('2011-01-25 11:20:33')];` |
| Логический | BOOL | `RETURN [true, false];` |
| Композитный | LIST | `RETURN [[1, 2], [4, 5]];` |

**Важное замечание о типах компонентов LIST**:

Neug поддерживает списки через типы данных tuple, что означает, что композитные типы могут быть гетерогенными. Вот несколько примеров:

Смешивание различных примитивных типов в одном списке:
```cypher
RETURN ['marko', 2];
```

Объединение различных типов свойств узлов в списке:
```cypher
MATCH (n:person) RETURN [n.name, n.age];
```

Поддержка вложенных структур списков:
```cypher
MATCH (n:person) RETURN [["name", n.name], ["age", n.age]];
```

**Ключевые технические детали**:
- Списки в Neug могут содержать элементы различных типов данных (гетерогенные списки)
- Это реализовано через внутреннюю поддержку типа данных tuple
- Преобразование типов выполняется автоматически, когда это возможно
- Вложенные списки полностью поддерживаются для сложных структур данных
- Система обеспечивает безопасность типов, сохраняя при этом гибкость в составлении списков

### Типы паттернов

#### NODE
- **Описание**: Представляет вершину в графе
- **Внутренняя структура**: Содержит `_ID` (внутренний идентификатор), `_LABEL` (метка узла) и поля свойств
- **Пример запроса**: `MATCH (n:person) RETURN n AS node_value;`
- **Формат Neug**: `{_ID: 0, _LABEL: person, id: 1, name: marko, age: 29}`

#### REL (Relationship)
- **Описание**: Представляет ребро в графе
- **Внутренняя структура**: Содержит `_SRC` (ID и LABEL исходного узла), `_DST` (ID и LABEL целевого узла), `_ID` (внутренний идентификатор связи), `_LABEL` (тип связи) и поля свойств
- **Пример запроса**: `MATCH ()-[r:knows]->() RETURN r AS rel_value;`
- **Формат Neug**: `{_ID: 2, _LABEL: knows, _SRC_LABEL: person, _DST_LABEL: person, _SRC_ID: 0, _DST_ID: 2, weight: 1.0}`

#### ПОВТОРЯЮЩИЙСЯ ПУТЬ (REPEATED PATH)
- **Описание**: Представляет собой путь, состоящий из повторяющихся рёбер в графе
- **Внутренняя структура**: Содержит каждый узел и связь вдоль пути, включая начальный и конечный узлы
- **Пример запроса**: `MATCH (a:person)-[p*1..2]->(c) RETURN p AS path_value;`
- **Формат Neug**: `{_ID: 0, _LABEL: person}, {_ID: 4294967298, _LABEL: created, _SRC_LABEL: person, _DST_LABEL: person, _SRC_ID: 0, _DST_ID: 2}, {_ID: 2, _LABEL: person}, {_ID: 4297064449, _LABEL: created, _SRC_LABEL: person, _DST_LABEL: software, _SRC_ID: 2, _DST_ID: 72057594037927937}, {_ID: 72057594037927937, _LABEL: software}`
