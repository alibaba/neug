# DML Clause

DML (Data Manipulation Language) предоставляет операции для вставки, удаления и изменения данных в графовых базах данных. Neug поддерживает как массовые операции с данными (например, COPY FROM), так и индивидуальные операции с данными (например, CREATE, SET и DELETE). В этом документе приведены примеры и объяснения для каждого типа операций.

## COPY FROM

Команда COPY FROM позволяет выполнять массовую загрузку данных из внешних источников данных и создавать вершины и ребра в хранилище графов. В настоящее время поддерживаются источники данных в формате CSV.

### Загрузка данных вершин

Загрузите данные вершин person из файла CSV. Каждая строка в CSV соответствует вершине, а столбцы — свойствам вершин, определенным в схеме person.

**person.csv:**
```
name,age
marko,39
vadas,27
josh,32
peter,35
```

**Command:**
```cypher
COPY person FROM "person.csv"
```

### Загрузка данных ребер

Load загружает данные ребер из CSV-файла. Первые два столбца указывают первичные ключи исходной и целевой вершины, а дополнительные столбцы определяют свойства ребер.

**knows.csv:**
```
src_name,dst_name,weight
marko,josh,1.0
marko,vadas,0.5
josh,peter,0.8
```

**Command:**
```cypher
COPY knows FROM "knows.csv"
```

## CREATE

Предложение CREATE используется для вставки новых вершин и связей в граф.

### Создание вершин

Создайте новые вершины с указанными свойствами. Если вершина с тем же первичным ключом уже существует, будет выведена ошибка.

```cypher
CREATE (a:person {name: 'taylor', age: 25}), (b:person {name: 'julie', age: 30})
```

### Создание вершин и связей

Создайте вершины и связи в одном выражении. Это удобно, когда нужно создать как сами вершины, так и связи между ними.

```cypher
CREATE (a:person {name: 'mars', age: 28})-[:knows {weight: 16.0}]->(b:person {name: 'jennie', age: 26})
```

### Создание связей между существующими вершинами

Сначала найдите существующие вершины, затем создайте между ними связь.

```cypher
MATCH (a:person {name: 'taylor'}), (b:person {name: 'julie'})
CREATE (a)-[:knows {weight: 20.0}]->(b)
```

## SET

Команда SET используется для обновления свойств существующих вершин и связей.

### Обновление свойств вершин

Обновите свойства конкретной вершины.

```cypher
MATCH (a:person)
WHERE a.name = 'marko'
SET a.age = 37, a.city = 'New York'
RETURN a.*
```

### Обновление свойств связей

Обновите свойства конкретной связи.

```cypher
MATCH (a:person)-[k:knows]->(b:person)
WHERE a.name = 'marko' AND b.name = 'josh'
SET k.weight = 10.0, k.since = '2023-01-01'
RETURN k.*
```

## DELETE

Команда DELETE используется для удаления вершин и связей из графа.

### Удаление вершин

Удалите вершину из графа. По умолчанию можно удалять только вершины, у которых нет связей, чтобы избежать появления висячих рёбер.

```cypher
MATCH (a:person)
WHERE a.name = 'marko'
DELETE a
```

### Удаление вершин со связями (DETACH DELETE)

Используйте DETACH DELETE, чтобы принудительно удалить вершину и все её связи. Это предотвращает ошибки при попытке удалить вершины, у которых есть существующие отношения.

```cypher
MATCH (a:person)
WHERE a.name = 'marko'
DETACH DELETE a
```

### Удаление связей

Удалите определённые связи между вершинами, сохранив сами вершины.

```cypher
MATCH (a:person)-[k:knows]->(b:person)
WHERE a.name = 'marko' AND b.name = 'josh'
DELETE k
```
