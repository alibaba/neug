# Клаузула MATCH

Клаузула `MATCH` используется для поиска паттернов в графовой базе данных. Она позволяет находить узлы, связи и пути, которые соответствуют определенным критериям.

## Поиск узлов

### Поиск узлов с одной меткой

Найти все узлы с определенной меткой. Этот запрос возвращает все узлы с меткой `person`.

```cypher
MATCH (p:person) RETURN p;
```

результат:
```
+-------------------------------------------------------+
| p                                                     |
+=======================================================+
| {_ID: 0, _LABEL: person, name: marko, age: 29} |
+-------------------------------------------------------+
| {_ID: 1, _LABEL: person, name: vadas, age: 27} |
+-------------------------------------------------------+
| {_ID: 2, _LABEL: person, name: josh, age: 32}  |
+-------------------------------------------------------+
| {_ID: 3, _LABEL: person, name: peter, age: 35} |
+-------------------------------------------------------+
```

### Поиск узлов с несколькими метками

Найдите узлы с любой из указанных меток. Этот запрос возвращает все узлы, помеченные как `person` или `software`.

**Примечание**: В отличие от Neo4j, Neug не поддерживает узлы с несколькими метками. В Neo4j запись `(p:person:software)` обозначает узлы, которые одновременно имеют обе метки — `person` и `software`. В Neug этот синтаксис означает объединение узлов, имеющих либо метку `person`, либо `software`.

```cypher
MATCH (p:person:software) RETURN p;
```

Результат:
```

+=============================================================================+
| {_ID: 0, _LABEL: person, name: marko, age: 29}                       |
+-----------------------------------------------------------------------------+

+-----------------------------------------------------------------------------+
| {_ID: 2, _LABEL: person, name: josh, age: 32}                        |
+-----------------------------------------------------------------------------+
| {_ID: 3, _LABEL: person, name: peter, age: 35}                       |
+-----------------------------------------------------------------------------+
| {_ID: 72057594037927936, _LABEL: software, name: lop, lang: java}    |
+-----------------------------------------------------------------------------+
| {_ID: 72057594037927937, _LABEL: software, name: ripple, lang: java} |
+-----------------------------------------------------------------------------+
```

### Поиск узлов с любым лейблом

Поиск узлов без указания лейбла. Neug поддерживает запросы без явного указания лейблов и автоматически выводит неизвестные лейблы во время компиляции на основе определенных ограничений схемы.

```cypher
MATCH (p) RETURN p;
```

результат:
```
+-----------------------------------------------------------------------------+

+=============================================================================+
| {_ID: 0, _LABEL: person, name: marko, age: 29}                       |
```

## Сопоставление узлов с условиями

В дополнение к ограничениям по меткам, вы можете указать условия фильтрации на основе свойств.

```cypher
MATCH (p:person {name: 'marko'}) RETURN p;
```

вывод:
```
+-------------------------------------------------------+
| p                                                     |
+=======================================================+
| {_ID: 0, _LABEL: person, name: marko, age: 29} |
+-------------------------------------------------------+
```

## Поиск повторяющихся путей

Neug поддерживает исследование путей переменной длины с повторениями, что является распространенной функцией в graph query.

### Поиск путей переменной длины

Найдите пути с переменным количеством hops (шагов). Этот запрос возвращает все пути, состоящие из 1 или 2 отношений.

```cypher
MATCH (p:person)-[k*1..2]->(f) RETURN k;
```

### Поиск путей с повторяющимися узлами и граничными условиями

Как указано в [спецификации Kuzu](https://docs.kuzudb.com/cypher/query-clauses/match/#filter-recursive-relationships), Neug также поддерживает фильтрацию свойств на каждом ребре в процессе поиска пути.

Этот запрос требует, чтобы каждое ребро в пути удовлетворяло условию `r.weight < 1.0`.

```cypher
MATCH (p:person {name: 'marko'})-[k:knows*1..2 (r, _ | WHERE r.weight <= 1.0)]->(f:person)
Return k;
```

### Поиск троп (Trail Path)

Используя опцию `TRAIL`, можно дополнительно ограничить повторяющиеся пути, гарантируя, что ни одно ребро не будет повторено. Это обеспечивает завершение итераций расширения пути без зацикливания.

```cypher
MATCH (p:person {name: 'marko'})-[k:knows* TRAIL 1..2]->(f:person)
Return k;
```

### Поиск простого пути

Используя опцию `SIMPLE`, можно дополнительно ограничить повторяющиеся пути, чтобы гарантировать, что ни один узел не будет повторяться — это обеспечит вывод только простых путей.

```cypher
MATCH (p:person {name: 'marko'})-[k:knows* ACYCLIC 1..2]->(f:person)
Return k;
```

### Поиск невзвешенного кратчайшего пути

Укажите опцию `SHORTEST`, чтобы получить невзвешенный кратчайший путь между двумя заданными узлами.

```cypher
MATCH (p:person {name: 'marko'})-[k:knows* SHORTEST 1..2]->(f:person {name: 'josh'})
RETURN k;
```

## Match Patterns

Команда `MATCH` поддерживает сложные паттерны поиска, которые комбинируют узлы, связи и условия различными способами для выражения сложных запросов к графу.

Ниже приведены некоторые классические паттерны графовых запросов, широко используемые в различных бенчмарках графовых запросов:
- Triangle Pattern
```
Match (a:person)-[:created]->(b:software),
      (c:person)-[:created]->(b:software),
      (a:person)-[:knows]->(c:person)
Where a.name <> b.name AND b.name <> c.name
Return count(*);
```

- Square Pattern
```
Match (a:person)-[:created]->(b:software),
      (c:person)-[:created]->(b:software),
      (a:person)-[:knows]->(d:person),
      (c:person)<-[:knows]-(d:person)
Where a.name <> b.name AND b.name <> c.name AND c.name <> d.name
Return count(*);
```

- Long Path
```
Match (a:person)-[:knows]->(b:person),
      (b:person)-[:knows]->(c:person),
      (c:person)-[:created]->(d:software),
      (d:software)<-[:created]-(e:person),
      (e:person)-[:knows]->(f:person)
Where a.name <> b.name AND b.name <> c.name 
    AND c.name <> d.name AND d.name <> e.name
Return count(*);
```

- Clique Path
```
Match (a:person)-[:created]->(b:software),
      (c:person)-[:created]->(b:software),
      (a:person)-[:knows]->(d:person),
      (c:person)<-[:knows]-(d:person),
      (a:person)-[:knows]->(c:person),
      (d:person)-[:created]->(b:software)
Where a.name <> b.name AND b.name <> c.name AND c.name <> d.name
Return count(*);
```

## Необязательное совпадение (Optional Match)

Команда `OPTIONAL MATCH` позволяет находить паттерны, которые могут присутствовать или отсутствовать в графе. Для частей паттерна, которые не были найдены, возвращается значение `null`.

Пример использования Optional Match:

```cypher
MATCH (a:person)-[:knows]->(b:person)
OPTIONAL MATCH (b:person)-[:created]->(c:software)
RETURN a.name, b.name, c.name
```

В результатах выше для каждой пары (a, b):
- Если у узла b есть связанные узлы c, возвращаются все тройки (a, b, c). Например, для ('marko', 'josh') соответствующими тройками будут {('marko', 'josh', 'lop'), ('marko', 'josh', 'ripple')}.
- Если у узла b нет связанных узлов c, для текущей пары (a, b) сохраняется строка с c=null. Например, для ('marko', 'vadas') результирующая тройка будет {('marko', 'vadas', null)}.

Это основная цель использования OPTIONAL MATCH — сохранить строки из основного MATCH, даже если дополнительный паттерн не совпадает.
