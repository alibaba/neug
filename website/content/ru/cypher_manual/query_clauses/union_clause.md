# Union Clause
Оператор Union в Neug используется для объединения результатов нескольких подзапросов в один общий набор данных. Все участвующие подзапросы должны возвращать согласованные схемы — то есть одинаковое количество колонок с совпадающими именами и типами данных.

На данный момент Neug поддерживает вариант `UNION ALL`, который объединяет результаты без удаления дубликатов. Доступны две синтаксические формы:
- **Standard Union**: аналог стандартного синтаксиса из [Kùzu](https://docs.kuzudb.com/cypher/query-clauses/union/).
- **Call Union**: расширенная форма, вдохновлённая [Neo4j](https://neo4j.com/docs/cypher-manual/current/subqueries/call-subquery/#call-post-union), позволяющая более гибко компоновать запросы.

## Стандартный Union

В стандартном использовании `UNION ALL` применяется для объединения результатов нескольких подзапросов. Union должен быть терминальным оператором, который объединяет выходные данные всех предыдущих ветвей.

```cypher
MATCH (n {name: 'marko'}) RETURN n.age
UNION ALL
MATCH (n {name: 'josh'}) RETURN n.age;
```

## Call Union

Вдохновленный [Neo4j](https://neo4j.com/docs/cypher-manual/current/subqueries/call-subquery/#call-post-union), Neug расширяет семантику union через блок `CALL {}` с параметризованным вводом, позволяя создавать более выразительные и модульные запросы. Эта конструкция позволяет:
- Выполнять дополнительную логику после объединения.
- Совместно использовать предварительно вычисленный контекст (например, связанные переменные) между ветвями union.

Пример:
```cypher
MATCH (person:person {id: 123})
WITH person
CALL (person) {
  MATCH (person)-[k:knows]->(friend)
  WHERE k.weight > 1.0
  RETURN friend

  UNION ALL

  MATCH (person)-[k:knows]->(friend)
  WHERE k.weight < 1.0
  RETURN friend
}
RETURN friend.id, friend.name

```

Этот запрос можно разбить на три этапа:
- **PreQuery**: Выполняется до блока `CALL {}` (например, MATCH (person)), формирует предварительно вычисленный контекст, который будет использоваться в подзапросах union.
- **Union Subqueries**: Определяются внутри блока `CALL {}`. Каждая ветвь имеет доступ к общему контексту (например, person).
- **PostQuery**: Выполняется после `CALL {}`, обрабатывает объединенный результат (например, RETURN friend.id, friend.name).

Синтаксис `CALL (person)` передает внешние переменные в область видимости union, позволяя каждому подзапросу получать доступ к общему контексту и работать с ним. Этот паттерн особенно полезен, когда нужно применить несколько стратегий фильтрации или обхода к одной и той же входной сущности.
