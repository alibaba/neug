# Клаузула Limit

Limit используется для управления количеством выводимых результатов. Помимо самостоятельного использования, она также может применяться вместе с Order BY для выполнения операций TopK. Neug в настоящее время поддерживает два типа выражений в Limit: 1. Одиночная целочисленная константа. 2. Арифметические выражения, содержащие только константные параметры.

## Limit с целочисленным значением

```
Match (a:person)
Return a.age
Limit 2;
```
Поскольку порядок вывода не задан, результатом могут быть любые две записи.

output:
```
+------------+
|   _0_a.age |
+============+
|         29 |
+------------+
|         27 |
+------------+
```

## Limit с целочисленным выражением

```
Match (a:person)
Return a.age
Limit 1+1;
```

output:
```
+------------+
|   _0_a.age |
+============+
|         29 |
+------------+
|         27 |
+------------+
```

# Skip Clause

Limit управляет количеством выходных результатов, что эквивалентно определению диапазона номеров строк выходного результата как [0, upper_bound). Skip управляет пропуском первых нескольких строк выходных результатов, что эквивалентно определению диапазона номеров строк выходного результата как [lower_bound, +∞), где мы предполагаем, что нумерация строк начинается с 0.

## Skip с целочисленным значением

```
Match (a:person)
Return a.age
Skip 2;
```
Запрос используется для пропуска первых двух строк результатов.

вывод:
```
+------------+
|   _0_a.age |
+============+
|         32 |
+------------+
|         35 |
+------------+
```

## Skip с целочисленным выражением

```
Match (a:person)
Return a.age
Skip 1+1;
```

вывод:
```
+------------+
|   _0_a.age |
+============+
|         32 |
+------------+
|         35 |
+------------+
```
