# Клаузула Order

Order используется для сортировки текущих результатов по свойствам, чтобы обеспечить детерминированный вывод. В настоящее время мы поддерживаем две опции сортировки: `ASC` для сортировки по возрастанию и `DESC` для сортировки по убыванию. Если не указано иное, по умолчанию используется сортировка по возрастанию. Стоит отметить, что Order можно использовать в комбинации с Limit, что эквивалентно операции TopK. Ниже мы рассмотрим распространенные паттерны использования.

## Сортировка по одному свойству

```
Match (a)
Return a.name
ORDER BY a.name ASC;
```

вывод:
```
+-------------+
| _0_a.name   |
+=============+
| josh        |
+-------------+
| lop         |
+-------------+
| marko       |
+-------------+
| peter       |
+-------------+
| ripple      |
+-------------+
| vadas       |
+-------------+
```

## Сортировка по нескольким свойствам

```
Match (a)-[b]->(c)
Return a.name, b.weight
ORDER BY a.name ASC, b.weight ASC;
```

результат:
```
+-------------+---------------+
| _0_a.name   |   _4_b.weight |
+=============+===============+
| josh        |           0.4 |
+-------------+---------------+
| josh        |           1   |
+-------------+---------------+
| marko       |           0.4 |
+-------------+---------------+
| marko       |           0.5 |
+-------------+---------------+
| marko       |           1   |
+-------------+---------------+
| peter       |           0.2 |
+-------------+---------------+
```

## Сортировка по выражениям

В дополнение к прямой сортировке по свойствам, ключ Order BY также может быть более сложным выражением, таким как результат арифметической операции, возвращаемое значение скалярной функции и т.д.

### Сортировка по предварительно вычисленному выражению

```
Match (a)-[b]->(c)
Return a.age, c.name
ORDER BY a.age + 10 ASC, c.name;
```

результат:
```
+------------+-------------+
|   _0_a.age | _2_c.name   |
+============+=============+
|         29 | josh        |
+------------+-------------+
|         29 | lop         |
+------------+-------------+
|         29 | vadas       |
+------------+-------------+
|         32 | lop         |
+------------+-------------+
|         32 | ripple      |
+------------+-------------+
|         35 | lop         |
+------------+-------------+
```

### Сортировка по результатам скалярной функции

```
Match (a)-[b]->(c)
Return a.name, c.name
Order BY label(a);
```

## Order with Limit

Кроме того, в сценариях BI (Business Intelligence) запросов, TopK является одной из самых распространенных операций, обрезая и выводя только самые значимые результаты. Neug также поддерживает такие запросы.

```
Match (a)-[b]->(c)
Return a.age, c.name
ORDER BY a.age + 10 ASC, c.name ASC
Limit 2;
``` 

вывод:
```
+------------+-------------+
|   _0_a.age | _2_c.name   |
+============+=============+
|         29 | josh        |
+------------+-------------+
|         29 | lop         |
+------------+-------------+
```
