# Клаузa Return

Return и With предоставляют схожий функционал, оба используются для дальнейшей агрегации или проекции данных. Разница в том, что Return должен возвращать и отображать обработанные результаты. Здесь мы не будем углубляться в детали функциональности Return, так как вы можете ознакомиться с примерами использования в разделе [With Section](with_clause.md). Мы сосредоточимся на выводе результатов и некоторых распространенных паттернах использования Return.

## Return Nodes

### Возврат узлов с одной меткой
```
Match (a:person) Return a;
```

Вывод показывает внутренний ID каждого узла person (назначенный графовой базой данных), метку и все свойства:
```
+-------------------------------------------------------+
| a                                                     |
+=======================================================+
| {_ID: 0, _LABEL: person, name: marko, age: 29} |
+-------------------------------------------------------+
| {_ID: 1, _LABEL: person, name: vadas, age: 27} |
+-------------------------------------------------------+
| {_ID: 2, _LABEL: person, name: josh, age: 32}  |
+-------------------------------------------------------+
| {_ID: 3, _LABEL: person, name: peter, age: 35} |
+-------------------------------------------------------+
```

### Возврат узлов с несколькими метками
```
Match (a) Return a;
```

Вывод показывает внутренний ID каждого узла, его метку и все свойства в собственном типе узла:
```
+-----------------------------------------------------------------------------+
| a                                                                           |
+=============================================================================+
| {_ID: 0, _LABEL: person, name: marko, age: 29}                       |
+-----------------------------------------------------------------------------+
| {_ID: 1, _LABEL: person, name: vadas, age: 27}                       |
+-----------------------------------------------------------------------------+
| {_ID: 2, _LABEL: person, name: josh, age: 32}                        |

| {_ID: 3, _LABEL: person, name: peter, age: 35}                       |
+-----------------------------------------------------------------------------+
```

## Return with TopK

Комбинация Return, OrderBy и Limit для вывода результатов TopK запроса
```
Match (a:person)-[:knows]->(b:person)
Return a.name, b.name
Order By a.name ASC, b.name ASC
Limit 2;
```

вывод:
```
+-------------+-------------+
| _0_a.name   | _2_b.name   |
+=============+=============+
| marko       | josh        |
+-------------+-------------+
| marko       | vadas       |
+-------------+-------------+
```
