# Функции графовых паттернов

В дополнение к различным операциям с функциями на основе реляционных данных, описанным ранее, Neug также поддерживает специальный набор функций для работы с данными узлов, ребер и путей, сгенерированными графовыми паттернами.

## Функции узлов

Функция | Описание | Пример
--------|----------|-------
ID() | Получить внутренний ID узла/ребра | Return (a) Return ID(a)
LABEL() | Получить метку узла/ребра | Match (a) Return LABEL(a)

## Функции ребер

В дополнение к функциям ID и LABEL существуют следующие операции с функциями на основе ребер:

Функция | Описание | Пример
--------|----------|-------
START_NODE() | Возвращает начальный узел данных ребра | Match ()-[b]->() Return START_NODE(b);
END_NODE() | Возвращает конечный узел данных ребра | Match ()-[b]->() Return END_NODE(b);

## Функции для работы с путями

Функция | Описание | Пример
---------|-------------|--------
NODES | Возвращает все узлы из рекурсивной связи | Match (a)-[b*2..3]->() Return NODES(b);
RELS | Возвращает все связи из рекурсивной связи | Match (a)-[b*2..3]->() Return RELS(b);
PROPERTIES | Возвращает заданное свойство из узлов/связей | Match (a)-[b*2..3]->() Return PROPERTIES(nodes(b), 'name'), PROPERTIES(rels(b), 'weight');
IS_TRAIL | Проверяет, содержит ли путь повторяющиеся связи | Match (a)-[b*2..3]->() Return IS_TRAIL(b);
IS_ACYCLIC | Проверяет, содержит ли путь повторяющиеся узлы | Match (a)-[b*2..3]->() Return IS_ACYCLIC(b);
LENGTH | Возвращает количество связей (длину пути) в рекурсивной связи | Match (a)-[b*2..3]->() Return LENGTH(b);
