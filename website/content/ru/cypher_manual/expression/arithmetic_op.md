# Арифметические операторы

Neug поддерживает распространенные арифметические операции, включая сложение (+), вычитание (-), умножение (*), деление (/) и остаток от деления (%).

## Тип результата

К арифметическим операциям могут применяться следующие типы данных: INT32, UINT32, INT64, UINT64, FLOAT, DOUBLE. Тип результата арифметических операций определяется типами операндов по следующим правилам:
- Если разрядность левого и правого операндов различается, результатом будет тип с большей разрядностью
- Если разрядности операндов совпадают, но знаковость различна, результатом будет следующий больший знаковый тип
- Максимальная разрядность для чисел с плавающей точкой — DOUBLE, для целых чисел — INT64

## Обработка ошибок

В дополнение к типам результатов, арифметические операции могут вызывать ошибки переполнения (overflow), антипереполнения (underflow) или деления на ноль (divide-by-zero) в зависимости от значений операндов. Для обработки этих ошибок Neug предоставляет различные подходы в зависимости от типа данных и режима сборки:

1. **Типы с плавающей точкой**: специальная обработка не выполняется; система использует стандартное поведение [IEEE 754](), возвращая значения Infinity, -Infinity или NaN.

2. **Целочисленные типы**: поведение отличается в режиме отладки (debug) и релизе (release):
   - **Debug mode**: допустимы более низкие требования к производительности, поэтому возможна валидация входных данных во время выполнения с явным выбрасыванием исключений
   - **Release mode**: высокие требования к производительности означают, что при переполнении/антипереполнении возвращаются неопределенные значения, за исключением деления на ноль, которое может привести к core dump

В следующей таблице описаны типы ошибок, которые могут возникнуть у каждого оператора:

| Оператор | Overflow | Underflow | DivideByZero | Пример |
|----------|----------|-----------|--------------|--------|
| +        | YES      | YES       | NO           | Return CAST(2147483647, 'int32') + CAST(1, 'int32') |
| -        | YES      | YES       | NO           | Return CAST(-2147483648, 'int32') - CAST(1, 'int32') |
| *        | YES      | YES       | NO           | Return CAST(2147483647, 'int32') * CAST(2, 'int32') |
| /        | NO       | NO        | YES          | Return 5 / 0 |
| %        | NO       | NO        | YES          | Return 5 % 0 |

## Арифметические операции с датами

Помимо числовых типов, арифметические операции также можно выполнять с типами datetime и interval. Neug поддерживает арифметические операции с датами и временем, которые позволяют добавлять или вычитать интервалы из дат и timestamp, а также вычислять разницу между временными значениями.

### Поддерживаемые операции

В следующей таблице описаны поддерживаемые операции арифметики дат:

| Операция | Описание | Пример | Результат |
|-----------|-------------|---------|--------|
| DATE + INTERVAL | Прибавить интервал к дате | `DATE('2011-02-15') + INTERVAL('5 DAYS')` | `DATE('2011-02-20')` |
| DATE - INTERVAL | Вычесть интервал из даты | `DATE('2011-02-15') - INTERVAL('5 DAYS')` | `DATE('2011-02-10')` |
| TIMESTAMP + INTERVAL | Прибавить интервал к временной метке | `TIMESTAMP('2011-10-21 14:25:13') + INTERVAL('30 HOURS 20 SECONDS')` | `TIMESTAMP('2011-10-22 20:25:33')` |
| TIMESTAMP - INTERVAL | Вычесть интервал из временной метки | `TIMESTAMP('2011-10-21 14:25:13') - INTERVAL('30 HOURS 20 SECONDS')` | `TIMESTAMP('2011-10-20 08:24:53')` |
| INTERVAL + INTERVAL | Сложить два интервала | `INTERVAL('5 DAYS') + INTERVAL('30 HOURS 20 SECONDS')` | `INTERVAL('6 DAYS 6 HOURS 20 SECONDS')` |
| INTERVAL - INTERVAL | Вычесть интервалы | `INTERVAL('5 DAYS') - INTERVAL('30 HOURS 20 SECONDS')` | `INTERVAL('3 DAYS 17 HOURS 39 MINUTES 40 SECONDS')` |
| TIMESTAMP - TIMESTAMP | Вычислить разницу во времени | `TIMESTAMP('2011-10-21 14:25:13') - TIMESTAMP('1989-10-21 14:25:13')` | `INTERVAL('8035 DAYS')` |
