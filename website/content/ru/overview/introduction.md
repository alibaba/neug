# Знакомство с NeuG

Добро пожаловать в NeuG (произносится как "нью-джи") — графовая база данных HTAP (Hybrid Transactional/Analytical Processing), предназначенная для высокопроизводительной встроенной графовой аналитики и обработки транзакций в реальном времени.

NeuG создана для решения растущей потребности в системах, которые могут одновременно справляться с операционными графовыми нагрузками и сложными аналитическими запросами к одним и тем же данным, предоставляя унифицированную платформу для современных графовых приложений.

## Что такое NeuG?

NeuG — это графовая база данных, которая объединяет лучшее из двух миров благодаря своей **уникальной архитектуре с двумя режимами работы**:
- **Транзакционная обработка (TP)**: операции с графами в реальном времени с гарантиями ACID
- **Аналитическая обработка (AP)**: высокопроизводительная аналитика графов и поиск паттернов
- **Два режима подключения**: гибкое развертывание в виде embedded-системы или сервиса
- **Оптимизированная производительность**: каждый режим настроен под свою целевую нагрузку

Основанная на современном ядре на C++, NeuG предоставляет:
- Модель данных Property Graph с языком запросов Cypher
- Столбцовое дисковое хранилище с сжатыми списками смежности
- Векторизованную и параллельную обработку запросов
- Управление транзакциями на основе MVCC

## Почему стоит выбрать NeuG?

### Уникальная архитектура с двумя режимами

Ключевая особенность NeuG — это **дву-режимная архитектура**, которая оптимизирует производительность под различные сценарии работы с графами:

#### Режим Embedded — Оптимизирован для аналитики (AP)
**Лучше всего подходит для**: сложного сопоставления паттернов, вычислений на полном графе и аналитических нагрузок
- **Пакетная загрузка данных**: эффективная обработка импорта больших объемов данных
- **Максимальная производительность запросов**: использование всех доступных ядер CPU для сложных запросов
- **Полное использование ресурсов**: каждый запрос может задействовать все системные ресурсы

**Типичные сценарии использования**:
- Аналитика графов и задачи data science
- Сложное обнаружение паттернов и алгоритмы на графах
- Пакетная обработка и ETL-операции
- Исследования и разведочный анализ данных

#### Режим сервиса - Оптимизирован для транзакций (TP)
**Лучше всего подходит для**: Приложений реального времени с частыми небольшими обновлениями и параллельным доступом
- **Параллельные операции чтения-записи**: Оптимизирован для высокочастотных, мелкомасштабных модификаций данных
- **Точечные запросы**: Эффективный доступ к небольшим подмножествам графовых данных
- **Поддержка нескольких сессий**: Обработка множественных параллельных подключений
- **Операции с низкой задержкой**: Быстрое время отклика для транзакционных рабочих нагрузок

**Типичные сценарии использования**:
- Системы рекомендаций в реальном времени
- Обнаружение мошенничества и управление рисками
- Обработка онлайн-транзакций

### Гибкое сочетание режимов

Настоящая мощность NeuG заключается в **гибком сочетании** обоих режимов:

- **Интеграция с Data Pipeline**: Используйте встроенный режим (embedded mode) для начальной загрузки больших объемов данных, затем переключайтесь в сервисный режим (service mode) для операционных запросов
- **Гибридные сценарии**: Экспортируйте контрольную точку (checkpointed data) из работающего TP-сервиса в отдельную копию для интенсивного AP-анализа
- **От разработки до продакшена**: Разрабатывайте и тестируйте с использованием встроенного режима, развертывайте как сервис для продакшена
- **Разделение рабочих нагрузок**: Запускайте AP-нагрузки на выделенных встроенных инстансах, при этом поддерживая TP-сервисы

> **Примечание**: Хотя встроенный режим может обрабатывать небольшие изменения, частые операции записи будут блокировать все запросы на чтение (и наоборот) из-за эксклюзивной блокировки. Аналогично, сервисный режим может выполнять большие паттерн-запросы, но не рекомендуется использовать все доступные потоки для ускорения одного запроса, так как это приведет к голоданию других параллельных запросов.

### Производительность и масштабируемость
NeuG разработан для высокопроизводительных графовых нагрузок с:
- Продвинутой оптимизацией запросов и планированием выполнения
- Векторизованными операциями для аналитических нагрузок
- Эффективным форматом хранения, оптимизированным для обхода графов
- Параллелизмом на нескольких ядрах для сложных запросов

### Гибкость и удобство использования
- **Кроссплатформенность**: Работает на Linux и macOS на архитектурах x86 и ARM
- **Простая интеграция**: Встроенная архитектура устраняет накладные расходы на управление сервером
- **Переключение режимов**: Бесшовный переход между встроенным и сервисным режимами

### Функции для корпоративного использования
- **ACID транзакции**: сериализуемые гарантии ACID для критически важных приложений
- **Параллельный доступ**: оптимизировано как для аналитики с преобладанием чтения, так и для операций с интенсивной записью
- **Обработка ошибок**: комплексные механизмы управления ошибками и восстановления
- **Мониторинг**: встроенный мониторинг производительности и профилирование запросов

NeuG разработан командой [GraphScope](https://graphscope.io), ведущей исследовательской группой из Alibaba, известной своим опытом в области графовых вычислений и обработки больших данных. Фреймворк интегрирует передовые методики, основанные на обширном опыте Alibaba в развертывании графовой аналитики для решения задач корпоративного масштаба в таких областях, как электронная коммерция, оптимизация цепочек поставок и обнаружение мошенничества.
