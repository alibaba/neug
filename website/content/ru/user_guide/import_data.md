# Импорт данных

NeuG поддерживает вставку данных в базу данных из указанного файла. Предварительным условием для вставки данных в базу данных является создание схемы графа, то есть структуры ваших таблиц узлов и связей.

Обычно вы можете использовать команду `COPY FROM`, чтобы импортировать данные из файла в базу данных.

## Копирование из CSV

### Импорт в таблицу узлов (Node Table)

Создайте таблицу узлов `person` следующим образом:

```cypher
CREATE NODE TABLE person(id INT64, name STRING, age INT64, PRIMARY KEY(id));
```

CSV-файл `person.csv` содержит следующие колонки:

```csv
id|name|age
1|marko|29
2|vadas|27
4|josh|32
6|peter|35
...
```

Следующий запрос импортирует данные из `person.csv` в таблицу `person`:

```cypher
COPY User FROM "user.csv" (header=true);
```

**Примечания:**

- Количество колонок в CSV-файле должно совпадать с количеством свойств, определённых для узла.
- Порядок колонок в CSV-файле должен соответствовать порядку свойств узла. Например, в приведённом выше примере свойства узла идут в порядке: `id`, `age`, `name`, что должно точно совпадать с порядком колонок в файле.

### Импорт в таблицу отношений
Создайте таблицу отношений `knows`, используя следующий запрос:
```cypher
CREATE REL TABLE knows(FROM person TO person, weight DOUBLE);
```
Файл CSV `person_knows_person.csv` содержит следующие столбцы:
```csv
person.id|person.id|weight
1|2|0.5
1|4|1.0
...
```
Следующий запрос импортирует файл `person_knows_person.csv` в таблицу `knows`.
```cypher
COPY knows from "person_knows_person.csv" (from="person", to="person", header=true);
```
**Примечание**
- При импорте в таблицу отношений NeuG предполагает, что первые два столбца в файле являются первичными ключами узлов `FROM` и `TO`; остальные столбцы соответствуют свойствам отношения.
- Параметры `FROM` и `TO` должны быть указаны в конфигурациях CSV, чтобы задать лейблы узлов `FROM` и `TO`.

### Конфигурации CSV
Поддерживаются следующие параметры конфигурации:
|Параметр|Описание|По умолчанию|
|---|---|---|
|`HEADER`|Является ли первая строка CSV-файла заголовком. Может быть true или false.|`false`|
|`DELIM` или `DELIMITER`|Символ, разделяющий различные столбцы в строках.|`\|`|
|`QUOTE`|Символ, начинающий строковую кавычку.|`"`|
|`ESCAPE`|Символ внутри строковых кавычек для экранирования QUOTE и других символов, например, разрыва строки.|`\`|
|`SKIP`|Количество строк, которые нужно пропустить из входного файла.|`0`|
|`PARALLEL`|Читать CSV-файлы параллельно или нет.|`true`|
|`IGNORE_ERRORS`|Пропускает некорректные строки в CSV-файлах, если установлено значение true.|`false`|
|`NULL_STRINGS`|Строки, которые должны рассматриваться как null в CSV-файле.|`""`(пустая строка)|
|`FROM`|Имя метки исходной вершины, используется только при импорте ребер|`-`|
|`TO`|Имя метки целевой вершины, используется только при импорте ребер|`-`|

**Примечание**
- Параметры должны быть указаны в виде пар ключ-значение, разделенных запятыми, например: `param1=value1, param2=value2, ...`
- Параметры нечувствительны к регистру, то есть `Header`, `HEADER` и `header` все допустимы и будут корректно распознаны в списке параметров.
- Для параметров булевого типа (например, `header`)：
  1. Используйте `True`, `true` или `1`, чтобы указать включенное состояние.
  2. Используйте `False`, `false` или `0`, чтобы указать выключенное состояние.
  3. Часть значения можно опустить, если это значение `true`, например, `header=true` можно сокращенно записать как `header`.
- `FROM` и `TO` - это специальная пара параметров. Эффективны только при импорте ребер из CSV-файла. Когда существуют несколько комбинаций меток для исходных и целевых вершин ребра (например, Comment-replyOf->Post, Comment-replyOf->Comment), необходимо указать `FROM` и `TO`. Использование этих параметров в других сценариях вызовет исключение.
