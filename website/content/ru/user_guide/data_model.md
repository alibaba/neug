# Модель данных

## Типы данных

`Neug` обеспечивает надежную поддержку широкого спектра типов данных, используемых в свойствах вершин и ребер. Эти типы данных организованы в пять различных категорий:

- **Примитивные типы:** Включают фундаментальные типы данных, такие как целые числа, числа с плавающей точкой и булевы значения, которые необходимы для представления базовых числовых и логических значений.

- **Строковые типы:** Эта категория включает типы, которые управляют текстовыми данными, позволяя эффективно хранить и обрабатывать строки внутри вершин и ребер.

- **Временные типы:** Эти типы предназначены для обработки информации о дате и времени, поддерживая различные представления времени для обеспечения временного анализа и запросов.

- **Типы массивов:** Типы массивов позволяют хранить коллекции элементов, обеспечивая представление списков или наборов значений, связанных с элементом графа.

- **Сложные типы:** (В настоящее время не поддерживаются) Могут включать пользовательские объекты или составные структуры, которые объединяют несколько примитивных типов, обеспечивая более сложное моделирование данных и взаимосвязей внутри графа.

### Типы данных времени

#### DATE

`DATE` обозначает календарный день, который характеризуется компонентами: год, месяц и день. Формат соответствует стандарту ISO-8601 (ГГГГ-ММ-ДД). Занимает 4 байта в хранилище.

```cypher
RETURN date('2022-06-06') as x;
```

#### DATE32

`DATE32` представляет календарный день как количество дней, прошедших с 1 января 1970 года. Дни до этой даты обозначаются отрицательными значениями. Занимает 4 байта в хранилище. (TODO: (xiaoli): убедиться, что работает)

```cypher
RETURN date32('2022-06-06') as x;
```

#### TIMESTAMP

`TIMESTAMP` объединяет компоненты даты и времени — час, минуту, секунду и миллисекунду — в формате стандарта ISO-8601 (YYYY-MM-DD hh:mm:ss[.zzzzzz][+-TT[:tt]]). Этот формат указывает дату (YYYY-MM-DD), время (hh:mm:ss[.zzzzzz]) и необязательное смещение времени [+-TT[:tt]]. Часть с датой обязательна, а временная часть необязательна, что позволяет включать миллисекунды [.zzzzzz] и смещение времени, если это необходимо. Занимает 8 байт в хранилище. Например: "1970-01-01 00:00:00.004666-10"

```cypher
RETURN timestamp("1970-01-01 00:00:00.004666-10") as x;
```

#### INTERVAL

`INTERVAL` представляет собой период времени, который может быть описан компонентами: год, месяц, день, час, минуты, микросекунды. Мы следуем определению Interval из [kuzu](https://docs.kuzudb.com/cypher/data-types/#interval).

```cypher
RETURN interval("1 year 2 days") as x;
```

### Типы массивов

Типы массивов в настоящее время не поддерживаются, но планируется добавить поддержку в ближайшем будущем.  
После добавления поддержки каждый элемент массива должен будет соответствовать одному из ранее упомянутых примитивных типов.

### Сложные типы

Сложные типы в настоящее время не поддерживаются, включая `STRUCT`, `MAP`, `UNION`, `BLOB`.

## Преобразование типов

Мы поддерживаем [приведение типов между числовыми типами](https://github.com/GraphScope/neug/issues/416).

- INT32
- UINT32
- INT64
- UINT64
- FLOAT
- DOUBLE

**Все пары этих типов могут быть преобразованы друг в друга**. Во время приведения типов компилятор выполняет проверки на переполнение, чтобы гарантировать безопасность преобразований. В таблице ниже приведены возможные сценарии переполнения, которые обрабатываются при этих преобразованиях:

| **INT32**  | ✅ Безопасно | ⚠️ Может произойти переполнение (если значение < 0)        | ✅ Безопасно | ⚠️ Может произойти переполнение (если значение < 0)        | ✅ Безопасно | ✅ Безопасно |
| **UINT32** | ⚠️ Может произойти переполнение (если значение > INT32_MAX)         | ✅ Безопасно | ✅ Безопасно | ✅ Безопасно | ✅ Безопасно | ✅ Безопасно |
| **INT64**  | ⚠️ Может произойти переполнение (если значение < INT32_MIN или > INT32_MAX) | ⚠️ Может произойти переполнение (если значение < 0 или > UINT32_MAX) | ✅ Безопасно | ⚠️ Может произойти переполнение (если значение < 0)        | ✅ Безопасно | ✅ Безопасно |
