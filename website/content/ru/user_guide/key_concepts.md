### Ключевые концепции
Database, Connection и Session — это фундаментальные концепции в NeuG.

## Database

**Database** в NeuG представляет собой граф, состоящий из вершин и рёбер. По умолчанию данные сохраняются на диске в указанной директории. NeuG также поддерживает режим **in-memory**, при котором данные хранятся только в памяти и не записываются на диск. Чтобы создать in-memory базу данных, установите `db_path` как пустую строку:

```python
from neug import Database

# Создание in-memory базы данных
db = Database(db_path="")
```

### Режимы доступа

Базы данных NeuG можно открывать в двух режимах: `read_write` и `read_only`.

- **read_write**: Открытие директории в режиме **read_write** предоставляет эксклюзивный доступ к ней для текущего процесса путем блокировки директории. Это предотвращает одновременный доступ к ней других процессов.
- **read_only**: При открытии в режиме **read_only** несколько процессов могут одновременно обращаться к директории в том же режиме **read_only**.

Важно отметить, что in-memory базы данных нельзя открыть в режиме `read_only`.

## Подключение

**Connection** — это основной интерфейс для взаимодействия с embedded базой данных. Он позволяет выполнять запросы и управлять данными.

### Установка соединений из базы данных

Объект **Connection** выступает в роли канала для доступа к базе данных, при этом его поведение зависит от режима доступа к базе данных.

- В режиме **read_write** Connection может выполнять как запросы на чтение (MATCH), так и запросы на запись (CREATE, COPY FROM). Для базы данных в режиме read_write может быть создан только один объект соединения, чтобы сохранить согласованность данных.
  
- В режиме **read_only** Connection ограничен выполнением запросов на чтение.

## Режим сервиса и сессия

NeuG также может работать в **режиме сервиса**, когда база данных функционирует как сервер и принимает удаленные подключения. Этот режим оптимален для приложений, требующих частого, параллельного или распределенного доступа.

Чтобы запустить NeuG в режиме сервиса, укажите хост и порт:

```python
import neug

# Запуск NeuG в качестве сервиса
```python
db = neug.Database("./neug/db")
service = db.serve(host="localhost", port=10000)
print(f"NeuG service started on {service}")
```

После запуска сервера клиенты могут подключаться с помощью объекта `Session`:

```python
from neug import Session

# Установка соединения с NeuG сервисом
session = Session("http://localhost:10000/")

# Выполнение запроса
session.execute('MATCH(n) RETURN count(n)')

session.close()
```

Можно создать несколько сессий в разных процессах для отправки параллельных запросов. Сервер базы данных гарантирует соблюдение ACID-свойств базы данных на протяжении всей работы.
